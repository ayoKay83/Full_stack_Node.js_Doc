JavaScript best practices

JavaScript was introduced in 1995 and has since become "the language of the web”.

The following are an attempt to explain some basic printiciples and touch upon best pratices.

Variables

Variables are symbolic names that creates a binding to a value. It dosn’t actually contain the value itself since you can have several variables that points to the same value. It’s just a representation. We can identify a variable by giving it a meaningful name also called a variable identifier.

let variableWithMeaningfulName = “Hello, I am a variable with a meaningful name”

let anotherVariableWithAMeaningfulName = "Hello, I am a variable with a meaningful name”

Declarations

Variables can be declared outside of functions which makes them global variables that can be accessed anywhere on the document/script. If you declare a variable inside a function it has a local scope and can only be accessed within that function. But local scope can either mean function scope or block scope depending on the type of declaration you use. When we declare a variable in JavaScript we can use 3 types of variable declaration:

var - declares a function scoped variable, optionally initializing it to a value that can change throughout the program.

let - declares a block scoped variable, optionally initializing it to a value that can change throughout the program.

const - declares a block scoped variable that dosen’t change throughout the program (it’s read-only).

Hoisting

Hoisting is JavaScripts behaviour of moving all declarations to the top of the scope. For this reason (while using var) it’s possible to use a variable before it is declared. However the initialization is not hoisted. 

 //this code will run and display undefined
  console.log(x)
  var x = 5;
  
  //JavaScript will hoist the declaration but NOT the initialization
  //and execute the code in the following order
  var x;  //declaration
  console.log(x);
  x = 5;  //initialization

Because x is displayed before it is initialized it will return undefined. Looking at the same code, if you use let to declare x it will result in a ReferenceError and declaring a const variable after it has been used results in a SyntaxError.

NB!To avoid issues with hoisting always declare your variables at the top of your scope or script.

Asynchronous operations

JavaScript is single-threaded but allows for asynchronous behaviour meaning that you can have several operations running in parallel. E.g. you run a function A and this function will start another function B before function A executes. Whenever Function B has finished it’s task it will notify function A and return the result (if any).
